Keyframes

Type Example
Array [ {value: 100, easing: 'easeOutExpo'}, {value: 200, delay: 500}, {value: 300, duration: 1000} ]

Keyframes are defined using an Array of property Object.
More details on Object values in the Specific property parameters.

If their is no duration specified inside the keyframes of one property, each keyframe duration will be equal to the animation total duration divided by the number of keyframes: 

anime({
  targets: '.el',
  translateX: [
    {value: 100}, // duration -> 2000 / 4 = 500 ms
    {value: 200}, // duration -> 2000 / 4 = 500 ms
    {value: 300}, // duration -> 2000 / 4 = 500 ms
    {value: 400} // duration -> 2000 / 4 = 500 ms
  ],
  duration: 2000
});

Timeline

Timeline basics

Timelines let you synchronise multiple animations together.
By default each animation added to the timeline will start after the previous animation ends.

Creating a timeline: 

var myTimeline = anime.timeline(parameters);
Argument Type Info Required
parameters Object The default parameters of the timeline inherited by children No

Adding animations to a timeline: 

myTimeline.add(parameters, offset);
Argument Types Info Required
parameters Object The child animation parameters, override the timeline default parameters Yes
time offset String or Number Check out the Timeline offsets section No

Time Offsets

Type Offset Example Infos
String '+=' '+=200' Starts 200ms after the previous animation ends
String '-=' '-=200' Starts 200ms before the previous animation ends
Number Number 100 Starts at 100ms, reagardless of the animtion position in the timeline

Time offset can be specified as a second optional parameter in the timeline .add() Function.
It defines when a animation starts in the timeline, if no offset is specifed, the animation will starts after the previous animation ends.
An offset can be relative to the last animation or absolute to the whole timeline.

no offset
relative offset ('-=600')
absolute offset (400)

Parameters inheritance

Write less code by defining default parameters.
Every timeline child inherits the default parameters. 

var myTimeline = anime({
  targets: '.el',
  easing: 'linear' // defines '.el' as the targets for each animation child
});

myTimeline
.add({
  translateX: 250 // animtate '.el' to 250px to the right with a 'linear' easing
})
.add({
  rotate: 180,
  easing: 'easeOutExpo' // override the default timeline easing
});

Play / Pause

animation.play();

Play a paused animation, or if the animation autoplay parameters is set to true/.

animation.pause();

Pause a running animation.

Restart

animation.restart();

Restart an animation from its initial values.

Reverse

animation.reverse();

Reverse the direction of an animation (running or not).

Seek

Jump to a specific time (in milliseconds).

Can also be used to control an animation while scrolling:

animation.seek((scrollPercent / 100) * animation.duration);

Timelines can be controled like any other anime instance.

States

Initialisation

States let you define and store multiple set of properties for one animation.

Add states using the states object:

var animation = anime({
  targets: '.el',
  easing: 'easeInOutCirc'
});

animation.states.stateName = {
  translateX: 250,
  duration: 1500,
  easing: 'easeOutExpo'
}

Then go directly to a specific state:

animation.states.goTo(stateName);

Or animate to a state with optional animation parameters:

animation.states.animateTo(stateName, animationParameters);

Default properties

The initial animation properties and parameters are inherited by all the animation states.

Defines default properties and parameters:

var animation = anime({
  targets: '.el',
  translateX: 0, // will be inherited by every states
  rotate: 0, // will be inherited by every states
  easing: 'easeInOutCirc'
});

animation.states.stateA = {
  translateX: 250
}

animation.states.stateB = {
  rotate: 45
}

Then go directly to a specific state:

animation.states.goTo(stateName);

Or animate to a state with optional animation parameters:

animation.states.animateTo(stateName, animationParameters);

Parameters inheritance

By default, only the states properties and parameters are animated and previous states changes are not overrided by the new state.

Defines default properties:

var animation = anime({
  targets: '.el',
  easing: 'easeInOutCirc'
});

animation.states.stateA = {
  translateX: 250
}

animation.states.stateB = {
  rotate: 45
}

Move the element by animating to state A:

animation.states.animateTo(stateA);

Then rotate the element by animating to state B.
Since state B doesn't override state A properties, the element will keep the initial transformation from state A. 

animation.states.animateTo(stateB);

CSS transforms order is relative to the order in which the states are called. If the new states apply a new transform proerty, the first animation will be canceled.

Callbacks & Promises

Update

Type Parameters Info
Function animation Return the current animation Object

update() callback is triggered on every frame as soon as the animation starts playing.

Begin

Type Parameters Info
Function animation Return the current animation Object

begin() callback is triggered when the animation.currentTime is equal or superior to the animation.delay.
It's also called if the animation duration is 0.

Run

Type Parameters Info
Function animation Return the current animation Object

run() callback is triggered on every frame if the animation.currentTime is equal or superior to the animation.delay.

Complete

Type Parameters Info
Function animation Return the current animation Object

complete() callback is triggered when the animation has finished.

Finished promise

animation.finished.then(function() {
  // Do things...
});

Every animation instances return a finished promise when the animation finised.

Promises are not suported in IE < 11.

SVG

Motion path

Animate an element relative to the x, y and angle values of an SVG path element. 

var myPath = anime.path('svg path');

The path function returns a new Function that returns the specified property:

Parameters Example Info
'x' myPath('x') Return the current x value in 'px' of the SVG path
'y' myPath('y') Return the current y value in 'px' of the SVG path
'angle' myPath('angle') Return the current angle value in 'degres' of the SVG path

Morphing

Creates transition between two svg shapes.

Shapes must have the same number of points!

Line drawing

Anime has a built in helper to create path drawing animation with the 'stroke-dashoffset' property.
Set the path 'dash-offset' value with anime.setDashoffset() in a From To formated value: 

anime({
  targets: 'svg path',
  strokeDashoffset: [anime.setDashoffset, 0]
});
Accepts Parameters Info
Function animation Return the current animation Object
